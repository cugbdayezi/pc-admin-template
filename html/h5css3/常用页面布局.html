4、清除浮动的方法(为什么要清浮动？请看图)

      
注意：当元素设置float浮动后，该元素就会脱离文档流并向左/向右浮动，直到碰到父元素或者另一个浮动元素， 浮动元素会造成父元素高度塌陷，所以当我们设置float后，需进行相应的清除浮动操作，清除方法：

(1)给父元素加高度包住子元素

问题：扩展性不好，在很多情况下，父元素的高度并不知道

(2)让父元素也进行浮动

问题：页面中所有元素都加浮动，margin左右自动失效（floats bad ！）,设置maring:0 auto;不能使元素左右水平居中

(3)在浮动元素后面添加一个空块级标签，并给空标签添加clear:both样式

问题：IE6 最小高度 19px；（解决后IE6下还有2px偏差），不要求兼容低版本浏览器可以使用此方法

(4)<br/>清浮动,在浮动元素后面添加<br/>

问题：不符合工作中：结构、样式、行为，三者分离的要求。

(5)after伪类清浮动方法(比较主流方法),例如给浮动元素的父级添加一个clear类样式


/*
 after伪类： 元素内部末尾添加内容；
	:after{content"添加的内容";} IE6，7下不兼容
 zoom 缩放 
 	a、触发 IE下 haslayout，使元素根据自身内容计算宽高。
	b、FF 不支持
 */
.clear:after{
  content: '';
  display:block;
  clear:both;
}
.clear{
  zoom:1;
}
